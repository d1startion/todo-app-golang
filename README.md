# REST API для управления задачами с авторизацией и слоистой архитектурой
## Архитектура

Проект построен по принципу разделения ответственности:

- **handler** — обрабатывает HTTP-запросы (парсинг, ответы)
- **service** — содержит бизнес-логику приложения
- **repository** — отвечает за работу с базой данных

Такой подход упрощает тестирование, поддержку и масштабирование приложения.

## Возможные улучшения

- Добавление пагинации для оптимизации работы с большими списками задач
- Внедрение структурированного логирования (slog)
- Реализация refresh токенов для повышения безопасности
- Добавление rate limiting для защиты API
